# Istio

Service mesh: Istiod control plane + Envoy sidecar data plane. Data-plane metrics are generated by the sidecars. Healthy: low error rate, bounded request duration, fast config push, no endpoint gaps.

Also load `envoy.md` for data-plane detail.

## Incident shapes

- **Inter-service 5xx spike** — `istio_requests_total{response_code="5xx"}` rises → usually target workload, not mesh
- **Config push lag** — `pilot_proxy_convergence_time` p99 spikes → sidecars slow to apply; stale routing
- **Endpoint discovery gap** — `pilot_endpoint_not_ready` / `dns_cluster_without_endpoints` nonzero → service has no healthy pods
- **Request duration regression** — `istio_request_duration_milliseconds` p95 climbs → downstream slow or mesh overhead
- **Sidecar down / unready** — traffic bypasses mesh or fails entirely

## Key metrics

| Metric | Unit | Signal |
|---|---|---|
| `istio_requests_total` | count | Per source/destination/response_code |
| `istio_request_duration_milliseconds` | ms | Latency histogram |
| `istio_request_bytes` / `response_bytes` | bytes | Payload sizes |
| `istio_request_messages_total` / `response_messages_total` | count | gRPC streaming |
| `pilot_endpoint_not_ready` | count | Endpoints not-ready at push time |
| `pilot_dns_cluster_without_endpoints` | count | Clusters with no resolved endpoints |
| `pilot_xds_pushes` | count | Config pushes |
| `pilot_push_triggers` | count | Push-trigger events |
| `pilot_proxy_convergence_time` | ms | Sidecar convergence latency |
| `pilot_proxy_queue_time` | ms | Queue before push |

## Derived signals

- Error rate by (source_workload, destination_workload) — localizes caller/callee pair.
- `p99 / p50` latency per destination — widening gap = long-tail issues.
- `convergence_time` p99 — high = control plane overloaded, stale config risk.
- `1 - (dns_cluster_without_endpoints / total_clusters)` — endpoint health fraction.

## Log patterns

Sidecar access-log flags — see `envoy.md` (`UH`, `NR`, `UC`, `UT`, `DC`).

Istiod:

- `Handled resource ... push` — config push events
- `push context pre-compute timed out` — control-plane overload
- `endpoint not ready` — discovery lagging
- mTLS STRICT mode denials — auth rejections

## Gotchas

- `istio_requests_total` cardinality is high (source × destination × namespace × response_code). Aggregate carefully.
- `reporter` label distinguishes sidecar: `source` = caller's sidecar, `destination` = callee's. 5xx with `reporter=source` only = target never received.
- mTLS PERMISSIVE → STRICT transitions can silently break non-mesh clients. Check `response_flags` for `AuthenticationPolicyRejected`.
- Gateway / VirtualService changes apply asynchronously; "new config not working" is often a convergence-time issue.
- Sidecar injection is namespace-controlled. Workloads without a sidecar don't appear in `istio_*` metrics — silently outside the mesh.
